# 🎵 AI音楽プロンプトジェネレーター（ローカルファーストPWA）

キーワード・ジャンル・ムードなどから、AI作曲ツール向けの**作曲プロンプト**と、ChatGPT/Claudeなど他のAIチャットボットにそのまま貼り付けられる**歌詞プロンプト**を生成するPWAアプリです。生成履歴・評価・プリセットはすべて端末内のIndexedDBに暗号化保存され、既定では一切外部へ送信されません。

以前のFlask版（`legacy/`）を全面的に置き換えた、React + Vite + TypeScript製の実装です。

## 主な機能

| 機能 | 説明 |
|---|---|
| 作曲プロンプト生成 | ジャンル・ムード・テンポ・ボーカル・楽器・構成・雰囲気を選択し、標準/詩的/ミニマルの3案（英語＋日本語）を生成 |
| 歌詞プロンプト生成 | 他のAIチャットボットに歌詞を書かせるための依頼文を生成。テーマキーワード・言語（日本語/英語）を指定可能 |
| プロンプトライブラリ | JSON/CSV（複数ファイル可）を取り込み、歌詞プロンプト生成時の参考テキストとして選択できる |
| 履歴・評価 | 生成結果を★評価・タグ付けして保存。全文検索・評価フィルタ・削除・暗号化JSONエクスポートに対応 |
| 自己学習ループ | 履歴の評価データから、よく高評価を得ている選択肢を自動的に上位表示（⭐マーク付き）。十分なデータが蓄積された組み合わせは提案として表示され、ユーザーが承認すると恒久的に優先表示されるようになる |
| Claude API連携（任意） | 歌詞プロンプトのみ、ユーザー自身のAPIキーでアプリ内から直接Claudeに送信し応答を確認できる（既定オフ、明示的な同意が必要） |
| PWA | manifest・Service Worker対応。ホーム画面への追加・オフライン起動が可能 |

## セットアップ

```bash
# Node.js 20+ を推奨
npm install
npm run dev
```

`http://localhost:5173` を開くと、初回起動時にパスフレーズの設定を求められます。このパスフレーズから端末内のデータを暗号化する鍵を導出するため、**忘れると保存済みデータは復元できません**（サーバー側にバックアップは一切ありません）。

### ビルド

```bash
npm run build    # dist/ に静的ファイルを出力
npm run preview  # ビルド結果をローカルで確認
```

### デプロイ

`dist/` は完全な静的ファイル一式なので、Vercel・Netlify・GitHub Pages等どの静的ホスティングにもそのままデプロイできます（サーバーサイドの処理は一切不要）。

- **Vercel**: リポジトリをインポートし、Root Directoryを`frontend`、Build Commandを`npm run build`、Output Directoryを`dist`に設定
- **Netlify**: Base directoryを`frontend`、Build commandを`npm run build`、Publish directoryを`frontend/dist`に設定

PWAとして正しく動作させるには、HTTPS配信（Vercel/Netlifyは既定で対応）が必要です。

## Claude API連携（任意機能）

「歌詞プロンプト」画面の各案に「Claudeに送信」ボタンがあります。これを使うには：

1. 設定タブで「送信同意を確認する」から同意する（既定は未許可）
2. 設定タブでAnthropicのAPIキーとモデルIDを登録する

APIキーはパスフレーズ由来の鍵でAES-GCM暗号化して端末内に保存されますが、**送信自体はブラウザから直接Anthropic APIへ行われる**ため、ブラウザの開発者ツール（ネットワークタブ）からはキーが見える構成です。これは個人利用のローカルツールという前提での設計判断であり、複数ユーザー向けにホスティングする製品でこのパターンをそのまま使うべきではありません。

作曲プロンプトは常にローカル生成のみで、外部送信は一切行われません。

## プライバシー・セキュリティ設計

- **ローカル優先**: 生成履歴・評価・プリセット・インポートしたプロンプトはすべてIndexedDBに保存。外部送信はClaude連携（任意・歌詞プロンプトのみ）を除き発生しない
- **暗号化**: パスフレーズ + PBKDF2（25万回・SHA-256）で鍵を導出し、AES-GCMで暗号化。導出した鍵はメモリ上のみに保持し、保存されるのはソルトのみ
- **自動マスキング**: 保存前にメールアドレス・電話番号・クレジットカード番号らしき文字列を自動的に`[MASKED_EMAIL]`等に置換
- **同意の記録**: 外部送信（Claude連携）は明示的な同意操作が必要で、同意日時を記録する
- **エクスポート**: 履歴・プリセット・ライブラリを暗号化されたJSONとしてエクスポート可能（設定はソルトごと含むため、同じパスフレーズがあれば復元できる）

## テスト

```bash
npm run test       # Vitestユニットテスト（生成エンジン・暗号化・マスキング・DBラッパー）
npm run test:watch # ウォッチモード
npm run test:e2e   # Playwright E2Eテスト（入力→生成→評価→履歴の一連。事前に `npx playwright install chromium` が必要）
```

## プロジェクト構成

```
frontend/
├── src/
│   ├── components/     # UIコンポーネント
│   ├── lib/             # 生成エンジン・DB・暗号化・マスキング・自己学習ロジック
│   ├── hooks/            # Reactカスタムフック
│   ├── types/            # 型定義
│   └── data/templates.json  # プロンプトテンプレのローカルコピー（同期元は ../prompts/templates.json）
├── e2e/                 # Playwright E2Eテスト
├── scripts/sync-templates.mjs  # ../prompts/templates.json をこの下にコピーするスクリプト
└── public/              # PWAアイコン・favicon
```

テンプレデータ（ジャンル・ムード等）の正本はリポジトリルートの`prompts/templates.json`です。編集したら`npm run sync:templates`でこのディレクトリ内のコピーに反映してください。

## 既知の制限

- パスフレーズを忘れた場合、データの復元手段はありません（意図的な設計）
- 自己学習ループは実装済みの短期（自動リランキング）・中期（提案の承認制）にとどまり、長期構想（複数端末のログを匿名化して集約しモデルを再学習する仕組み）は、サーバー集約が前提となるためこのローカル完結アプリのスコープ外としています
- Claude連携はブラウザから直接APIを呼ぶ設計のため、個人利用を前提としています
